# Product

## Register

product

## Users

Club players, improving beginners, intermediate players, amateur competitors, junior players and parents, coaches, and advanced players. They often use the app one-handed on a phone while saving a tutorial, reviewing practice material, or returning to a skill at the table.

## Product Purpose

Turn scattered table-tennis tutorial videos into an organized personal learning library of skills, topics, notes, timestamped takeaways, drills, mistakes, and learning paths. Success means capture is fast, organization can happen later, learning material is easy to revisit, and sharing remains explicit and private by default.

## Brand Personality

Clear, practical, focused, coach-like, lightweight, and learning-oriented. The interface should feel like a dependable mobile learning notebook, with the graph quietly powering useful relationships.

## Anti-references

Do not resemble a generic bookmark manager, heavy CMS, social media feed, graph database console, generic note app, or AI-first product. Avoid desktop layouts compressed into a phone, decorative dashboard cards, and navigation that hides the core capture workflow.

## Design Principles

1. Capture first, classify later.
2. Put the next learning or practice action within thumb reach.
3. Make persistence, offline, install, and sharing states honest.
4. Lead with table-tennis learning language, not storage or graph terminology.
5. Preserve the same route hierarchy across phone, tablet, and desktop.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Support 320 CSS px reflow, safe areas, software keyboards, reduced motion, visible focus, semantic landmarks, and minimum 44 by 44 CSS px touch targets. Never rely on color or hover alone, and keep manual URL capture available regardless of install or native-share capability.
